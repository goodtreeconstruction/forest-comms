# MEMORY.md - Long-Term Memory

## Forest Chat Rules (Team Workflow)

### Message format
- Use tags:
  - **[TASK]** new task assignment
  - **[Q]** question needing answer
  - **[STATUS]** progress update
- **One topic per message** (keep threads clean/parseable).

### Task ownership & tracking
- **Single owner per task** (set a primary to avoid duplicate effort).
- **Clear done criteria** (define what “finished” means).
- **Use task IDs** and **close explicitly** when complete.
- If using a Kanban/board: move tasks **To Do → In Progress → Done** as state changes.

### Response discipline (anti-loop)
- **No auto-replies** unless directly asked or @mentioned.
- When assigned a task: **acknowledge**, then **wait for explicit “LFG / start”** before doing work.
- **Max 1 follow-up question per turn**; if blocked, state **blocker + options**.
- **Loop guard:** if there are **2 back-and-forths without new info**, stop and propose a concrete next action.

### Efficiency
- **Use tools only when needed**; batch tool calls.
- Don’t repeat the same check within **30 minutes** unless requested/needed.
- Status updates should be **3 bullets max**: **Done / Next / Blocked**.
